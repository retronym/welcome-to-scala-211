Reactive Building Block: Functional Programming

  • Favour immutability
  • Lambdas to glue apps together (guided by types!)
  • Uniform API for FP on collections and Futures (learn once, code anywhere!)

Reactive Building Block: Futures

  • Start with j.u.c.Future
  • + non-blocking composition
  • scala.concurrent.Future or j.u.c.CompletableFuture
  • related term: Promise

Reactive Building Block: Actors

  • Encapsulates state
  • Only contactable via its message queue
  • Lightweight (can have millions in a JVM)
  • Organized in a supervisor heirarchy (policy based failure recovery)
  • Configurable dispatchers and mailboxes
